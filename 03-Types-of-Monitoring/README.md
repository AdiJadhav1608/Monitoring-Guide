# 🚀 Types of Monitoring

---

## 📌 Introduction

Monitoring is not just one thing — it includes **different types**, each focusing on a specific part of the system.

👉 To fully understand system behavior, we use **multiple monitoring types together**

---

## 🎯 Major Types of Monitoring

---

### 🖥️ 1️⃣ Infrastructure Monitoring

This type focuses on **hardware and system resources**.

#### 📊 What it monitors:
- CPU usage 🔥  
- Memory usage 🧠  
- Disk usage 💾  
- Network traffic 🌐  

#### 🧠 Example:
- Server CPU reaches 90% → System slows down

👉 Helps ensure **servers are healthy**

---

### 🌐 2️⃣ Application Monitoring

This focuses on **application performance and behavior**

#### 📊 What it monitors:
- Response time ⏱️  
- Request rate 📈  
- Error rate ❌  
- API performance 🔗  

#### 🧠 Example:
- API taking 5 seconds instead of 200ms → Performance issue

👉 Helps improve **user experience**

---

### 📜 3️⃣ Log Monitoring

This involves analyzing **logs generated by systems and applications**

#### 📊 What it monitors:
- Error logs ❌  
- Access logs 🔐  
- System events ⚙️  

#### 🧠 Example:
- "Database connection failed" error in logs

👉 Helps in **debugging and troubleshooting**

---

### 🔍 4️⃣ Network Monitoring

This focuses on **network performance and connectivity**

#### 📊 What it monitors:
- Bandwidth usage 📡  
- Packet loss 📉  
- Latency ⏱️  

#### 🧠 Example:
- High latency → Slow communication between services

👉 Ensures **smooth communication between systems**

---

### 👤 5️⃣ User Experience Monitoring

Also known as **End-User Monitoring (EUM)**

#### 📊 What it monitors:
- Page load time 🌐  
- User interactions 🖱️  
- Real user performance 📊  

#### 🧠 Example:
- Website loads slowly for users in a specific region

👉 Focuses on **real user experience**

---

### 🤖 6️⃣ Synthetic Monitoring

This uses **simulated requests** to test system behavior

#### 📊 What it monitors:
- Website uptime 🌍  
- API availability 🔗  
- Login functionality 🔐  

#### 🧠 Example:
- Bot checks website every 1 minute

👉 Helps detect issues **before users face them**

---

### ☁️ 7️⃣ Cloud Monitoring

This focuses on **cloud-based infrastructure and services**

#### 📊 What it monitors:
- Virtual machines 🖥️  
- Storage usage 💾  
- Cloud services health ☁️  

#### 🧠 Example:
- Auto-scaling triggers when traffic increases

👉 Essential for **cloud environments (AWS, Azure, GCP)**

---

## 🧩 Combined Monitoring Approach

👉 In real-world systems, we combine all types:

```text
Infrastructure + Application + Logs + Network + User Monitoring
                    = Complete Visibility 🔍
```

---

## ⚙️ Example: Network Monitoring Script

```bash
#!/bin/bash

# This script checks if a website is reachable

WEBSITE="google.com"

ping -c 2 $WEBSITE > /dev/null

if [ $? -eq 0 ]; then
    echo "✅ $WEBSITE is reachable"
else
    echo "❌ $WEBSITE is not reachable"
fi
```

### 📖 Explanation:
- `ping` → Checks network connectivity  
- `-c 2` → Sends 2 packets  
- `$?` → Stores last command status  
- 0 = Success, non-zero = Failure  

👉 Simple **network monitoring check**

## 🧪 Interview Questions

### ❓ What are the types of monitoring?

Infrastructure, Application, Log, Network, User Experience, Synthetic, and Cloud monitoring.

---

### ❓ What is the difference between application and infrastructure monitoring?

Application monitoring focuses on app performance, while infrastructure monitoring focuses on system resources like CPU and memory.

---

### ❓ What is synthetic monitoring?

It uses automated scripts or bots to simulate user actions and test system availability.

---

## 🚀 Summary

- Monitoring has multiple types 🧩  
- Each type focuses on different system aspects  
- Combining all gives complete system visibility 🔍  
- Helps improve performance, reliability, and user experience  

👉 **Right monitoring = Strong and stable system**
